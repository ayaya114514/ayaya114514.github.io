### 1. [CollaboratoR: A scalable workflow for collaborative data entry and management](http://arxiv.org/abs/2606.19280v1)

**作者**：Patrick Bills, Ashwini Ramesh, Lais Petri, Alejandra Martinez Blancas, Kelly Kapsar, Amar Deep Tiwari 等  
**发表日期**：2026-06-17  
**分类**：q-bio.QM  
**Paper ID**：`arxiv:2606.19280v1`

#### 核心贡献
本文提出 CollaboratoR，一个用于协作式数据录入、验证、聚合和版本追踪的 R workflow/package，目标是在 ad-hoc spreadsheets 和复杂数据系统之间提供轻量方案。两个 case studies 显示，自动 validation 能提前发现格式和录入问题，提升大型协作数据集的 traceability 并减少后期清洗。

#### 技术栈
- **语言/框架**：R
- **模型/算法**：Automated data validation、data aggregation、FAIR data workflow
- **关键库与工具**：Google Sheets、GitHub
- **数据集**：plant competition 数据库、avian interaction 数据库两个 case studies
- **开源情况**：摘要提到 R package 和 GitHub version control，但未给代码链接

#### 上手难度
- **前置知识**：R、协作数据录入、数据验证、FAIR data principles、GitHub 基础
- **算力需求**：低至中等：普通 8–16 核 CPU、16–32GB RAM 可运行单样本或交互式分析；批量 workflow 建议 32–64GB RAM 和任务队列，无需专用 GPU（推算，依据：数据库查询/生信 pipeline）
- **复现成本**：容易，核心是 R package + Google Sheets/GitHub workflow，但具体安装链接未在摘要中给出
- **适合谁读**：本科生、硕博生、研究数据管理人员、做 meta-analysis 的研究员

#### 一句话总结
给协作数据录入加自动校验和版本控制。

- **语言/框架**：未提及
- **模型/算法**：Stochastic SIR、multi-type branching likelihood、ordinary differential equations、latent internal branching times
- **关键库与工具**：未提及
- **数据集**：模拟 outbreaks；印度 Karnataka COVID-19 contact-tracing data
- **开源情况**：未提及

#### 上手难度
- **前置知识**：传染病动力学、SIR model、likelihood inference、branching process、contact tracing data
- **算力需求**：中等：单次模型求解预计用 8–32 核 CPU、32–64GB RAM；参数扫描或 ensemble simulation 建议 HPC 并行，通常耗时数小时至数天，GPU 不是必需（推算，依据：动力学/PDE/随机模拟）
- **复现成本**：中等偏困难，需要实现 likelihood/ODE 并获得 transmission tree 数据
- **适合谁读**：研究员、硕博生、流行病建模和 phylodynamics 方向读者
